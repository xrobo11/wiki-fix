안내서 100% 활용하기

안녕하세요, 여러분!
이 문서는 XROBO를 여행하는 여러분을 위한 안내서입니다.
여기엔 XROBO 익스트림 로봇을 조립하고 코딩하는 방법이 모두 담겨져 있습니다.
XROBO 익스트림 로봇을 만드는 데 매뉴얼이 도움이 될 거예요.
그럼, 이제부터 로봇을 만들어볼까요?

꼭 읽어주세요!

많이 하는 실수/질문 TOP5

가장 많이 발생한 실수와 질문 상위 5개를 골랐습니다.
로봇을 조립하고 코딩하기 전 반드시 확인해주세요!

⚠️ 많이 하는 실수/질문 TOP5
    앵글을 풀 때 너무 많이 풀지 말아주세요.
    앵글을 풀 때는 볼트를 시계 반대 방향으로 5~6번만 돌려주세요. 볼트를 너무 많이 풀면 앵글이 분리될 수 있어요.
    만약 앵글이 분리되었다면 '앵글&프레임 조립 분해 방법'를 참고하여 앵글을 조립해주세요.
    배터리를 끼울 때 배터리 방향을 한번 더 확인해주세요.
    배터리 케이스에 배터리를 끼울 때는, 스프링이 있는 쪽에 배터리의 평평한 쪽이 오도록 끼워주세요.
    배터리를 잘못 끼우면 배터리 케이스가 고장날 수 있어요.
    배터리케이스를 하나만 사용할 때, CPU보드의 12V 전원 커넥터에 점퍼를 꼭 끼워주세요.
    배터리케이스를 하나만 사용할 때는 CPU보드의 12V 전원 커넥터에 점퍼를 끼워주세요. 점퍼가 끼워져 있지 않으면 CPU보드의 전원이 켜지지 않아요.
    만약 점퍼를 잃어버렸다면 3x8 볼트를 커넥터에 끼워주세요.
    배터리케이스를 두 개 사용할 때는 점퍼를 빼고 사용해야 합니다.
    X1 단계는 코딩을 하지 않아도 돼요.
    X1 단계는 CPU보드에 예시 프로그램이 내장되어 있어 코딩을 하지 않아도 로봇을 동작시킬 수 있어요.
    직접 코딩을 하고 싶은 분들만 mBlock 코딩툴을 설치하여 코딩해주세요.
    X1보다 높은 단계를 하다가 다시 내려온 경우에도 코딩이 필요합니다.
    리모컨 페어링은 주변에 아이디가 겹치는 다른 로봇이 있을 때만 해주세요.
    리모컨을 사용하려면 CPU보드가 리모컨 아이디를 받아오는 '페어링' 과정이 필요해요.
    그러나 구매한 리모컨은 기본적으로 CPU보드와 이미 페어링되어 있으므로 따로 페어링 해줄 필요가 없어요.
    페어링은 주변에 다른 로봇이 있을 때만 필요해요.
    따라서 로봇을 혼자 사용하는 경우 로봇과 리모컨 전원을 켜기만 하면 바로 리모컨으로 로봇을 조종할 수 있어요.

앵글&프레임 조립 분해 방법

앵글과 프레임을 조립하거나 분리하는 방법이 설명되어 있습니다.

  • 앵글과 프레임 연결하기
  • 앵글과 프레임 분리하기
  • 분해된 앵글 조립하기

중 자신에게 맞는 설명을 골라 읽어주세요.

✔️ 앵글과 프레임 연결하기
  1. 앵글 안쪽 돌기에 프레임의 구멍을 맞춰 끼워주세요.
  2. 십자 드라이버로 앵글의 볼트를 시계 방향으로 돌려 볼트를 조여주세요. 앵글이 프레임을 꽉 잡아주면서 프레임이 고정됩니다.
  3. 드라이버로 볼트를 조일 때 드라이버가 헛돌면 볼트가 마모됩니다. 심하게 마모된 경우 앵글을 사용할 수 없게 되니, 드라이버가 헛돌지 않게 적당한 힘으로 볼트를 조여주세요.
✔️ 앵글과 프레임 분리하기
  1. 십자 드라이버로 앵글의 볼트를 반시계 방향으로 돌려 볼트를 풀어주세요. 앵글이 헐렁해지면서 프레임이 앵글과 프레임이 분리됩니다.
  2. 드라이버로 볼트를 풀 때 드라이버가 헛돌면 볼트가 마모됩니다. 심하게 마모된 경우 앵글을 사용할 수 없게 되니, 드라이버가 헛돌지 않게 적당한 힘으로 볼트를 풀어주세요.
✔️ 분해된 앵글 조립하기
  1. 앵글 상판과 하판을 맞춘 후, 앵글 구멍에 스프링을 끼운 다음 볼트를 넣습니다.
  2. 십자 드라이버로 볼트를 시계 방향으로 돌려 볼트를 조여주세요. 앵글을 조이는 것과 똑같이 조여주면 됩니다.
  3. 볼트를 너무 꽉 조이지 말고 볼트 머리가 앵글 위쪽에 조금 튀어나올 정도로 조여주세요. 이렇게 해야 나중에 프레임을 끼울 때 편리합니다.

배터리 사용 방법

배터리와 배터리케이스를 사용하는 방법이 설명되어 있습니다.

  • 배터리 끼우기
  • 배터리케이스 조립하기
  • 배터리케이스 CPU보드에 연결하기

중 자신에게 맞는 설명을 골라 읽어주세요.

✔️ 배터리 끼우기
  1. 배터리를 4개 준비해주세요.
  2. 배터리 케이스에서 배터리를 끼우는 곳을 보면 스프링이 있는 쪽과 없는 쪽이 있습니다. 둘 중 스프링이 있는 쪽에 배터리의 평평한 면이 오도록 끼워주세요.
  3. 배터리 방향을 반대로 연결하면 CPU보드의 전원이 켜지지 않습니다.
✔️ 배터리케이스 조립하기
  1. 배터리케이스를 덮개로 덮습니다. 덮개 옆면의 홈으로 배터리케이스의 케이블이 빠져나와야 합니다.
  2. 배터리케이스를 밑판에 끼웁니다. 덮개 양 끝의 튀어나온 부분이 밑판의 홈에 들어가야 배터리케이스가 제대로 조립됩니다.
✔️ 배터리케이스 CPU보드에 연결하기
  1. 배터리케이스를 하나만 쓰는 경우
    1. CPU보드 12V 전원 커넥터에 점퍼를 연결해줍니다. (점퍼가 없다면 3x8 볼트를 커넥터에 끼웁니다.)
    2. 배터리케이스 케이블을 CPU보드의 6V 전원 커넥터에 연결합니다.
  2. 배터리케이스를 두 개 쓰는 경우
    1. 배터리케이스 케이블을 각각 CPU보드의 6V, 12V 전원 커넥터에 연결해줍니다.

케이블 연결 방법

하드웨어에 케이블을 연결하는 방법이 설명되어 있습니다.

각 부품별 자세한 연결 방법을 살펴보고 싶다면 '하드웨어 매뉴얼'을 참고해주세요.

✔️ 케이블 연결하기
  1. 3-PIN 케이블을 준비합니다.
    • 3-PIN 케이블은 검은 선 한 개, 하얀색 선 두 개로 이루어져 있습니다.
  2. 하드웨어의 핀에 케이블을 끼웁니다.
    1. 캡이 있는 하드웨어의 경우: 하드웨어의 캡을 살펴보면 특정 핀을 가리키고 있는 '화살표' 표시가 있습니다. 이곳에 케이블의 검은색 선이 오도록 케이블을 끼웁니다.
    2. 캡이 없는 하드웨어의 경우: 하드웨어의 핀이 있는 부분을 살펴보면 특정 핀의 뒤쪽 프레임이 '하얀색'으로 표시되어 있습니다. 이곳에 케이블의 검은색 선이 오도록 케이블을 끼웁니다.

mBlock 설치 방법

mBlock은 XROBO의 코딩툴입니다.

직접 코딩을 하거나, 웹사이트에서 예시 프로그램을 다운받아 사용할 때 mBlock을 필요합니다.

여기에는 직접 코딩한 프로그램 또는 예시 프로그램을 로봇의 CPU보드로 업로드하는 방법이 설명되어 있습니다.

  • mBlock 설치가 필요한 경우/필요 없는 경우
  • mBlock 설치하기

중 자신에게 맞는 설명을 골라 읽어주세요.

혹시 mBlock 설치가 필요한지, 필요하지 않은지 잘 모르겠다면 반드시 'mBlock 설치가 필요한 경우/필요 없는 경우'를 읽어주세요.

✔️ mBlock 설치가 필요한 경우/필요 없는 경우
  1. mBlock 설치가 필요한 경우
    • 프로그램을 직접 코딩했을 때
    • 홈페이지에서 다운받은 예시 프로그램을 사용할 때
  2. mBlock 설치가 필요 없는 경우
    • 코딩을 하지 않는 X1 단계(예시 프로그램이 이미 업로드 되어 있음)
✔️ mBlock 설치하기
  1. 다운로드 링크로 이동해주세요.
  2. 'mBlock PC version'을 찾아, 'Download'를 클릭해주세요.
  3. 설치 파일이 다운 되면 실행시켜 설치를 시작해주세요.
  4. 'install'을 클릭하고 설치될 때까지 기다려주세요.
  5. 설치에 성공하면 '확인' 창이 뜹니다. '확인'을 클릭해주세요.
    1. 만약 설치에 실패한 경우, 'uninstall'을 눌러 프로그램을 삭제한 후 다시 다시 설치를 시도해주세요(4번으로 가주세요.)
  6. '마침' 버튼을 클릭해주세요.
  7. 바탕 화면을 살펴보면 mBlock 바로가기 버튼이 있습니다. mBlock을 실행해주세요.
  8. '장치 추가' 버튼을 눌러 장치 목록을 열어주세요.
  9. XROBO를 찾아 클릭하여 다운로드 합니다.
    1. 별을 클릭하면 즐겨찾기 설정이 가능합니다.
  10. 이제 코딩을 해주세요.

단계별 동작 예시 프로그램 사용 방법

XROBO에선 코딩이 어려운 친구들을 위해 익스트림 단계별로 '예시 프로그램'을 제공하고 있습니다.

웹사이트에서 예시 프로그램을 컴퓨터에 다운받고, 그 프로그램을 다시 CPU보드에 업로드해주면 쉽게 로봇을 사용해볼 수 있습니다.

여기에는 예시 프로그램을 웹사이트에서 다운받아 mBlock에서 불러오는 방법이 설명되어 있습니다.

✔️ 예시 프로그램 다운받아 불러오기
  1. 다운로드 링크로 이동해주세요.
  2. 원하는 익스트림 단계를 찾아 'download' 파일을 클릭해주세요.
  3. 방금 다운 받은 파일을 압축 해제합니다.
    1. 다운로드 받은 파일은 주로 '다운로드' 폴더에서 확인할 수 있습니다.
    2. 폴더를 클릭한 후 창 상단을 살펴보면 '압축 해제' 버튼이 있습니다. 이를 클릭하여 파일을 압축 해제 할 수 있습니다.
  4. mBlock를 실행하여 상단 메뉴바에서 파일>내 컴퓨터 가져오기를 클릭합니다.
  5. 압축 해제한 파일을 선택한 후, '열기'를 클릭합니다.
  6. 코드가 열립니다.

프로그램 업로드 방법

코딩을 마친 후에는 코딩한 프로그램을 CPU보드로 옮겨주는 작업이 필요합니다.

이렇게 프로그램을 CPU보드로 옮기는 것을 '업로드'라고 말합니다

여기에는 직접 코딩한 프로그램 또는 예시 프로그램을 로봇의 CPU보드로 업로드하는 방법이 설명되어 있습니다.

  • 업로드가 필요한 경우/필요 없는 경우
  • 업로드 준비물
  • 프로그램 업로드하기

중 자신에게 맞는 설명을 골라 읽어주세요.

✔️ 업로드가 필요한 경우/필요 없는 경우
  1. 업로드가 필요한 경우
    • 프로그램을 직접 코딩했을 때
    • 홈페이지에서 다운받은 예시 프로그램을 사용할 때
  2. 업로드가 필요 없는 경우
    • 코딩을 하지 않는 X1 단계(예시 프로그램이 이미 업로드 되어 있음)
✔️ 업로드 준비물
  1. 다운로드 케이블
  2. 다운로드 보드
✔️ 프로그램 업로드하기
  1. CPU보드의 다운로드 포트에 다운로드 보드를 연결합니다.
  2. 다운로드 케이블을 다운로드 보드에 끼웁니다.
  3. 케이블의 나머지 한쪽을 컴퓨터의 USB 포트에 끼웁니다.
  4. mBlock에서 '연결'을 클릭합니다.
  5. '장치 리스트 보기'를 클릭합니다.
  6. 장치를 선택하고 '연결' 버튼을 누릅니다.
    1. 장치 리스트에서 장치의 이름은 모두 'COM'과 숫자의 조합으로 뜨기에, 장치가 여러개 있는 경우 그 중 CPU보드가 무엇인지 정학히 알기 어렵습니다. 이땐 다운로드 케이블을 CPU보드에서 뺐다가 다시 끼워줍니다. 리스트에서 사라졌다가 다시 추가된 장치가 CPU보드입니다.
  7. 업로드 버튼을 눌러 코드를 업로드합니다.

프로그램 실행 방법

로봇에 업로드된 프로그램을 실행시키는 방법이 설명되어 있습니다.

  • 직접 코딩한 프로그램을 사용하는 경우 (RF AUTO 모드)
  • 예시 프로그램을 사용는 경우 (SCHOOL 모드)

중 자신에게 맞는 설명을 골라 읽어주세요.

✔️ 직접 코딩한 프로그램을 사용하는 경우 (RF AUTO 모드)
  1. CPU보드의 전원 버튼을 눌러 전원을 켭니다.
  2. START 버튼을 짧게 누르면(1초 미만) 업로드한 코드가 바로 실행됩니다.
    1. START 버튼을 길게 누르면 리모컨 페어링 모드로 넘어가게 됩니다. 이때는 전원을 껐다 켜주세요.
✔️ 예시 프로그램을 사용는 경우 (SCHOOL 모드)
  1. CPU보드의 전원 버튼을 눌러 전원을 켭니다.
  2. START 버튼을 길게 누르면(2초 이상) 프로그램 선택 모드가 켜집니다.
  3. START 버튼을 짧게 눌러 프로그램을 선택합니다. 1번 프로그램부터 시작해서 버튼을 누를 때마다 차례대로 2번, 3번 프로그램으로 바뀌었다가 마지막에는 다시 1번으로 돌아옵니다.

    점퍼 캡이 있다면 12V 전원 커넥터에 점퍼 캡을 연결해주세요.
    점퍼 캡을 갖고 있지 않은지 확인해주세요.

    배터리가 연결되어 있는지 확인해주세요.

    배터리를 CPU보드에 제대로 연결해주세요.

    배터리케이스를 한 개만 사용할 때, 12V 전원 커넥터에 점퍼 캡이 꽂혀있지 않다면 전원 스위치를 눌러도 전원이 켜지지 않아요.

    배터리를 올바른 방향으로 끼웠는지 확인해주세요.
    배터리케이스에 배터리가 잘못 끼워진 경우 CPU보드의 전원 스위치를 눌러도 CPU보드에 빨간 불만 들어오고 전원이 켜지지 않아요.
    배터리케이스의 스프링에 배터리의 평평한 부분이 오도록 끼워졌는지 확인해주세요.
    배터리가 충분한지 확인해주세요.
    배터리가 부족한 경우 CPU보드의 전원 스위치를 눌러도 전원이 켜지지 않아요.
    배터리를 새 걸로 교체해주세요.
    문제가 해결되지 않을 때
    배터리케이스 케이블이 단선되었거나
    배터리케이스의 전원 커넥터 부분가 고장났거나
    CPU보드의 전원 커넥터 부분이 고장났을 수 있습니다.

    (X-키패드를 사용하는 경우) X-키패드 버튼을 누를 때 CPU보드가 리셋되는지 확인해주세요.

    X-키패드 1번 버튼을 누르면 CPU보드가 리셋될 수 있습니다.
    X-키패드 선 연결을 확인해주세요.
    배터리케이스 케이블이 제대로 연결되었는지 확인해주세요.
    로봇이 충격을 받은 경우 CPU보드에 연결되어 있던 배터리케이스가 CPU보드에서 빠질 수 있습니다.
    또한 배터리가 CPU보드에 제대로 연결되지 않은 경우 CPU보드에 전원이 연결되었다 끊기길 반복하며 CPU가 반복적으로 리셋됩니다.
    배터리가 충분한지 확인해주세요.
    하드웨어를 작동시킬 때 하드웨어가 순간적으로 많은 전압을 사용하게 되는데, 이때 배터리가 부족하면 CPU보드가 리셋됩니다.
    배터리를 새 걸로 교체해주세요.

LED

⚠️ LED가 안 켜질 때
    LED가 CPU의 OUT포트에 올바르게 연결되어 있는지 확인해주세요.
    LED를
    • 잘못된 OUT포트에 연결한 경우
    • OUT포트가 아니라 IN포트에 연결한 경우
    • 포트에 케이블을 잘못 연결한 경우
    LED가 작동하지 않습니다.

    LED 케이블을 올바른 OUT포트에 세로 방향으로, 화살표에 까만색 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    LED 핀에 케이블이 제대로 연결되어 있는지 확인해주세요.
    LED 핀에 케이블이 잘못 연결된 경우 LED가 동작하지 않을 수 있습니다.

    LED의 화살표에 케이블의 까만색 선이 오도록 선을 연결해주세요. LED 핀은 3개씩 2줄이 있는데, 어떤 줄에 연결하는지는 상관 없습니다.
    그렇게 연결할 경우, 케이블의 까만 선이 LED 뒷 판의 하얗게 표시된 부분에 연결됩니다.
    코딩이 제대로 되어 있는지 확인해주세요.
    LED 상태를 OFF로 잘못 설정하거나, LED를 제어할 OUT포트를 잘못 코딩하거나, LED를 블록을 두 개 연달아 코딩하거나, LED 온오프 블록이 아닌 다른 코딩 블록을 잘못 사용하는 등의
    문제가 있을 경우 LED가 작동하지 않습니다.
    코딩을 다시 한 번 확인해주세요.
    문제가 해결되지 않는다면
    케이블이 단선 되었거나, OUT포트에 문제가 생겼거나, LED 자체에 문제가 생겼을 수 있습니다.
⚠️ LED가 계속 켜져있거나 제멋대로 동작할 때
    LED가 CPU의 OUT포트에 올바르게 연결되어 있는지 확인해주세요.
    LED가 다른 출력 하드웨어를 제어하는 OUT포트에 잘못 연결한 경우, LED가 계속 켜져있을 수 있습니다.

    LED 케이블을 올바른 OUT포트에 세로 방향으로, 화살표에 까만색 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    코딩이 제대로 되어 있는지 확인해주세요.
    LED 상태를 ON로 잘못 설정하거나, LED를 제어할 OUT포트를 잘못 코딩하거나, LED 온오프 블록이 아닌 다른 코딩 블록을 잘못 사용하는 등의
    문제가 있을 경우 LED가 작동하지 않습니다.
    코딩을 다시 한 번 확인해주세요.
    문제가 해결되지 않는다면
    OUT포트에 문제가 생겼거나, LED 자체에 문제가 생겼을 수 있습니다.

DC모터

⚠️ DC모터가 아예 안 움직일 때
    CPU에 드라이브 보드가 꽂혀있나요?
    CPU에 드라이브 보드가 꽂혀있나요?
    DC모터의 작동을 도와주는 드라이브 보드가 없는 경우 DC모터가 작동하지 않습니다.
    CPU보드에 드라이브 보드의 화살표 방향이 CPU 안쪽을 향하도록 연결해주세요.
    CPU의 드라이브 보드가 제대로 된 위치에 연결되어 있나요?
    DC모터를 코딩할 때 M1, M2 포트를 사용하도록 코딩했다면 M1, M2 포트가 있는 쪽에 드라이브 보드를 연결하고,
    M3, M4 포트를 사용하도록 코딩했다면 M3, M4 포트가 있는 쪽에 드라이브 보드를 연결해야 합니다.
    드라이브 보드가 잘못 연결된 경우 DC모터가 작동하지 않습니다.
    드라이브 보드를 바른 위치에 연결해주세요.
    CPU에 드라이브 보드가 정확한 방향으로 연결되어 있나요?
    드라이브 보드가 위아래가 뒤집힌 채로 CPU에 연결된 경우 DC모터가 작동하지 않고 발열이 발생합니다.
    드라이브 보드의 화살표 방향이 CPU의 안쪽을 향하도록 CPU에 연결해주세요.
    DC모터 케이블이 CPU의 모터 포트에 정확하게 연결되어 있나요?
    DC모터의 케이블이 모터 포트에 가로 방향으로 연결되어 있거나, 두 개의 핀 중 한 핀에만 연결되어 있는 경우
    DC모터가 작동하지 않습니다.
    DC모터 케이블을 모터 포트에 세로 방향으로, 화살표 방향에 모터 케이블의 까만 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    휠(바퀴)이 갈리진 않았나요?
    휠을 DC모터와 연결할 때, 휠의 연결부가 갈린 경우 DC모터가 정상적으로 작동하는데도 DC모터가 움직이지 않는 것처럼 보일 수 있습니다.
    DC모터에서 휠을 분리하여 연결부가 갈리지 않았는지 확인해주세요.
    연결부가 갈린 경우 연결부의 모양이 축의 형태와 같은 육각형이 아니라 원에 가깝습니다.
    코딩이 제대로 되어 있나요?
    모터 속도 또는 시간을 잘못 코딩하거나, 모터 포트를 잘못 코딩하거나, 모터 코딩 블록 위치가 잘못된 경우
    DC모터가 작동하지 않을 수 있습니다.
    코딩이 제대로 됐는지 다시 한번 확인해주세요.
    배터리가 충분한가요?
    배터리가 부족한 경우 DC모터가 작동하지 않을 수 있습니다.
    배터리를 새 걸로 교체해주세요.
    그래도 해결되지 않는다면
    DC모터가 고장났거나, DC모터가 연결된 모터 포트가 고장났을 수 있습니다.
⚠️ DC모터가 반대로 회전할 때
    DC모터 케이블을 정확한 모터 포트에 연결했나요?
    M1, M3에 연결해야 할 DC모터를 M2, M4에 연결하거나, M2, M4에 연결해야야 할 DC모터를 M1, M3에 연결할 경우 DC모터가 반대로 회전합니다.

    DC모터를 정확한 모터 포트에 연결해주세요.
    DC모터 선이 CPU의 모터 포트에 정확한 방향으로 연결되어 있나요?
    DC모터 케이블이 모터 포트에 반대로 연결되면 모터가 반대로 회전합니다.
    DC모터 케이블을 모터 포트에 세로 방향으로, 화살표 방향에 모터 케이블의 까만 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    코딩이 제대로 되어 있나요?
    모터 속도에 (-)를 붙이면 안 되는데 잘못 붙이거나, (-)를 붙여야 하는데 떼버리면 모터가 반대로 회전합니다.
    자동차 로봇의 좌회전, 우회전을 코딩할 때 이런 실수가 자주 발생합니다.
    코딩된 모터 속도를 다시 한번 확인해주세요.
⚠️ DC모터 속도가 갑자기 느려지거나 갑자기 움직이지 않을 때
    배터리가 충분한가요?
    배터리가 부족한 경우 DC모터의 속도가 느려지거나, DC모터가 아예 움직이지 않습니다.
    배터리를 새 걸로 교체해주세요.

서보모터

⚠️ 서보모터가 안 움직일 때
    서보모터 케이블이 정확한 OUT포트에 바르게 연결되어 있나요?
    서보모터를 코딩한 OUT포트가 아닌 다른 OUT포트에 연결하거나, OUT포트가 아니라 IN포트에 연결하거나, 포트에 케이블을 잘못 연결한 경우 서보모터가 작동하지 않을 수 있습니다.
    서보모터 케이블을 올바른 OUT포트에 세로 방향으로, 화살표에 까만색 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    코딩이 제대로 되어 있나요?
    서보모터를 제어할 OUT포트를 잘못 코딩하거나, 서보모터 블록을 두 개 연달아 코딩하는 등 코딩에 문제가 있을 경우 서보모터가 움직이지 않습니다.
    코딩을 다시 한 번 확인해주세요.
    배터리가 충분한가요?
    배터리가 부족한 경우 서보모터가 움직이지 않습니다.
    배터리를 새 걸로 교체해주세요.
    서보혼이 갈리진 않았나요?
    서보혼을 서보모터와 연결할 때, 서보혼의 연결부가 갈린 경우 서보모터가 정상적으로 작동하는데도 서보모터가 움직이지 않는 것처럼 보일 수 있습니다.
    서보모터에서 서보혼을 분리하여 연결부가 갈리지 않았는지 확인해주세요.
    연결부가 갈린 경우 서보혼의 연결부 모양이 축의 형태와 같은 육각형이 아니라 원에 가깝습니다.
    문제가 해결되지 않는다면
    서보모터를 OUT포트에서 빼고 손으로 돌렸을 때 잘 돌아가는지 확인해주세요.
    잘 돌아간다면, 배터리가 부족하거나, CPU의 OUT포트에 문제가 생겼거나, 서보모터 케이블이 단선되었을 수 있습니다.
    만약 특정 방향으로 돌아가지 않는다면, 서보모터가 잘못 조립되었을 수 있습니다.
    축이 아예 돌아가지 않는다면, 서보모터의 기어에 문제가 생겼을 수 있습니다.
⚠️ 서보모터가 생각한 것과 다르게 움직일 때
    서보모터 케이블이 정확한 OUT포트에 바르게 연결되어 있나요?
    서보모터를 코딩한 OUT포트가 아닌 다른 OUT포트에 연결한 경우 서보모터가 생각한 것과 다르게 작동할 수 있습니다.
    예를 들어 LED를 연결해야 할 OUT포트에 서보모터를 잘못 연결하는 경우, 서보모터가 이상하게 동작합니다.
    서보모터 케이블을 올바른 OUT포트에 세로 방향으로, 화살표에 까만색 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    코딩이 제대로 되어 있나요?
    서보모터 각도를 잘못 코딩한 경우 서보모터가 생각한 것과 다르게 작동할 수 있습니다.
    서보모터 각도를 제대로 코딩했는지 확인해주세요.
    서보모터가 잘못 조립되었을 수 있습니다.
    서보모터를 조립할 때 서보모터 축의 홈 위치와, 서보혼 홈 위치를 같게하여 조립했는지 확인해주세요.
    서보모터에 프레임을 연결할 때도 두 개의 홈 위치를 일치시킨 후 조립해야 합니다.
⚠️ 서보모터가 갑자기 느려졌거나 아예 움직이지 않을 때
    케이블이 제대로 연결되어 있나요?
    동작 중 로봇이 충격을 받아 서보모터 케이블이 빠졌을 수 있습니다.

    서보모터 케이블이 CPU의 OUT포트에 연결되어 있는지 확인해주세요.

    서보모터 케이블이 짧아 다른 3-PIN 케이블을 추가로 연결했다면 해당 선도 잘 연결되어 있는지 확인해주세요.
    배터리가 충분한가요?
    배터리가 부족한 경우 서보모터가 움직이지 않습니다.
    배터리를 새 걸로 교체해주세요.

리모컨

⚠️ 페어링이 안 될 때
    CPU보드와 리모컨에 RF모듈이 연결되어 있나요?
    CPU보드와 리모컨에 RF모듈이 빠져있는 경우 페어링이 되지 않습니다.
    CPU보드는 RF모듈이 없으면 START 버튼을 길게 눌렀을 때 CPU보드가 빠르게 깜박거리는 페어링 모드로 전환되지 않습니다.
    RF모듈을 연결해주세요.
    배터리가 충분한가요?
    로봇 또는 리모컨의 배터리가 부족할 경우 페어링이 안 될 수 있습니다.
    배터리를 새 걸로 교체해주세요.
    문제가 해결되지 않는다면
    리모컨의 펌웨어가 삭제됐을 수 있습니다. A/S를 맡겨주세요.
⚠️ 로봇이 멋대로 동작할 때
    리모컨으로 로봇을 조종하던 중 리모컨 전원이 꺼졌나요?
    로봇을 조종하던 중 리모컨 전원이 꺼지면 로봇이 마지막 동작 상태 그대로 계속 움직입니다.
    이때 리모컨을 다시 켜고 리모컨 버튼을 다시 눌렀다가 손을 떼면 로봇이 멈춥니다.
    주변에 아이디가 같은 리모컨이 있나요?
    내가 사용하는 리모컨과 다른 사람의 리모컨 아이디가 같은 경우 다른 리모컨으로도 내 로봇을 조종할 수 있습니다.
    이때 두 사람 중 한 명이 리모컨 페어링을 통해 리모컨 아이디를 바꿔주면 문제가 해결됩니다.
    코딩이 제대로 되었나요?
    코딩이 잘못된 경우, 로봇이 멋대로 동작할 수 있습니다.
    특히 DC모터 시간 블록과 속도 설정 블록을 바꿔 사용하는 실수가 잦습니다.
    코드를 다시 한 번 확인해주세요.
    코드가 제대로 업로드 되었나요?
    코드가 업로드 되지 않아 리모컨이 작동하지 않는 경우가 있습니다.
    코드를 다시 한 번 업로드 해주세요.
⚠️ 모든 버튼이 다 안 될 때/아무 반응이 없을 때
    CPU보드와 리모컨에 RF모듈이 연결되어 있나요?
    CPU보드와 리모컨에 RF모듈이 빠져있는 경우 리모컨 버튼을 사용할 수 없습니다.
    CPU보드는 RF모듈이 없으면 START 버튼을 길게 눌렀을 때 CPU보드가 빠르게 깜박거리는 페어링 모드로 전환되지 않습니다.
    RF모듈을 연결해주세요.
    코딩이 제대로 되었나요?
    주로 '계속 반복하기' 블록이 없을 때 이런 문제가 발생합니다.
    '계속 반복하기' 블록이 있는지 확인해주세요.
    또한 코드의 다른 부분에 문제가 없는지도 확인해주세요.
    코드가 제대로 업로드 되었나요?
    코드가 업로드 되지 않은 경우 리모컨이 작동하지 않을 수 있습니다.
    코드를 다시 업로드해주세요.
    페어링이 제대로 되어 있나요?
    리모컨이 페어링되지 않은 경우 리모컨이 작동하지 않을 수 있습니다.
    리모컨을 다시 페어링해주세요.
    문제가 해결되지 않는다면
    RF모듈이 고장났을 수 있습니다.
    CPU의 RF모듈이 고장난 경우, CPU에 RF모듈이 연결되어 있더라도 START 버튼을 길게 눌렀을 때 CPU보드가 빠르게 깜박거리는 페어링 모드로 전환되지 않습니다.
    RF모듈을 교체하거나 A/S를 맡겨주세요.
⚠️ 특정 버튼만 안 될 때
    코딩이 제대로 되었나요?
    특정 버튼을 눌렀을 때 어떤 동작을 할지 코딩되지 않았거나 잘못 코딩되었을 수 있습니다.
    코딩을 다시 한 번 확인해주세요.
    리모컨 버튼 눌렀을 때 리모컨의 파란색 LED가 켜지나요?
    특정 리모컨 버튼을 눌렀을 때 리모컨의 파란색 LED가 켜지지 않는다면 버튼이 고장난 것입니다.
    다른 버튼을 사용하거나 A/S를 맡겨주세요.
    LED가 고장났을 수도 있습니다.
    문제가 해결되지 않는다면
    특정 버튼을 눌렀을 때 작동해야 할 하드웨어 자체에 문제가 생겼을 수 있습니다.
    해당 하드웨어가 CPU보드에 잘 연결되어 있는지, 하드웨어가 고장난 건 아닌지 확인해주세요.

X키패드

⚠️ 버튼을 눌렀는데 작동하지 않을 때
    X-키패드가 올바른 IN포트에 올바르게 연결되어 있나요?
    X-키패드를 코딩한 IN포트가 아닌 다른 IN포트에 연결하거나, IN포트가 아니라 OUT포트에 연결하거나, 포트에 케이블을 잘못 연결한 경우 X-키패드가 작동하지 않습니다.
    X-키패드 케이블을 올바른 IN포트에 세로 방향으로, 화살표에 까만색 선이 오도록(까만 선이 CPU 바깥 방향을 향하도록) 연결해주세요.
    X-키패드 핀에 케이블이 제대로 연결되어 있나요?
    X-키패드에 케이블을 잘못 연결하거나, 연장선을 잘못 연결한 경우 X-키패드가 동작하지 않을 수 있습니다.
    X-키패드 선 연결을 확인해주세요.
    버튼을 눌렀을 때 X-키패드의 LED가 켜지나요?
    버튼을 눌렀을 때 X-키패드의 LED가 켜지지 않는다면 X-키패드의 버튼이 고장난 것입니다.
    이 경우 고장나지 않은 버튼을 사용하거나, 다른 X-키패드를 사용하거나, A/S를 맡겨주세요.
